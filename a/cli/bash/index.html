ヒアドキュメント

※POSIX標準ではないか。少なくともkshでは動かなさそう。

Examples:

cat << EOS > cat.out
foo
  bar
    baz
EOS

BODY="$(cat << EOS
Dear Ken,

I'm so happy to write this message.
I'll be back soon!
EOS
)"

ヒアドキュメントは文字列リテラルではなく、標準入力らしい。

参考:

  • bashのヒアドキュメントを活用する - Qiita

正規表現

POSIXか、EREなのだと思う。

SYNOPSIS:

if [[ "$str" =~ ^ba*r$ ]]; then
  echo match
end

## マッチしないことの検査
if [[ ! "$str" =~ ^ba*r$ ]]; then
  echo not match
end
